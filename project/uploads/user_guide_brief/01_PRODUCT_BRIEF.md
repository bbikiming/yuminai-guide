# 01 — 제품 브리프 (한 페이지)

> 디자이너가 5분만에 "이 앱이 뭔지" 잡을 수 있도록.

## 이름

**Yuminai (유미나이)**

## 1줄 정의

> **바이브 코딩을 위한 1인 macOS 워크스페이스.**
>
> Claude Code · OpenAI Codex · Obsidian Vault · Telegram · Git을 한 화면에서 오케스트레이션해
> *사고 → 코드 → 문서화 → 공유* 사이클을 끊김 없이 잇는 SwiftUI 네이티브 앱.

## 한 단락 (가이드 도입부용)

Yuminai는 코딩 에이전트(Claude Code, Codex CLI)를 GUI 채팅 인터페이스로 감싼 macOS 데스크톱 앱입니다. 터미널 안에 갇혀 있던 코딩 세션을 사이드바·인스펙터·텔레그램으로 확장해, **다중 워크스페이스**를 동시에 관리하고, **Obsidian 노트**를 컨텍스트로 자동 주입하고, **모바일에서도 명령**을 보낼 수 있게 만들었습니다. Claude Code가 추론 엔진이고, Yuminai는 그 위의 통합·UI 레이어입니다.

## 비전 — 3년 후

> 본인의 모든 *코딩 사이클*이 Yuminai 한 화면에서 시작·끝나도록 만든다.
> 외부 도구는 통합되어 있고, 컨텍스트는 자동으로 따라온다.
> 모바일에서 명령을 띄우면 책상 앞 macOS가 받아 작업을 진행한다.

## 핵심 가치 (3개)

### 1. 컨텍스트 무중단성 (Context Continuity)
Obsidian 노트, 이전 세션, 코드베이스 상태가 **자동으로** 작업 컨텍스트로 들어간다.
매번 복사·붙여넣기 안 한다. `@note-name`으로 inline 삽입, 세션은 SwiftData에 영속.

### 2. 모바일 ↔ 데스크톱 연속성 (Mobile ↔ Desktop)
Telegram으로 명령을 띄우면 데스크톱이 받아서 진행하고 결과를 알린다.
책상 떠나도 작업이 멈추지 않는다. 양방향 — 알림 받고, 추가 명령 보내고, 작업 결과 받고.

### 3. 하네스 가능성 (Composable Harness)
사용자(본인)가 자기 워크플로우에 맞는 **rules · agents · skills · hooks**를
워크스페이스에 끼워넣는다. claude-forge 패턴을 GUI로.

## 정체성 — *무엇이 아닌가*

| 아닌 것 | 이유 |
|---------|------|
| Cursor / Windsurf 같은 IDE | 코드 편집은 별도 IDE/에디터가 담당. Yuminai는 *대화형 작업 셸* |
| 일반 LLM 채팅 앱 (ChatGPT 데스크톱 등) | 바이브 코딩 워크플로우(파일/git/외부 통합)가 1급 |
| 팀 협업 도구 | 1인용. 멀티유저 기능 없음 |
| 새 LLM 추론 엔진 | Claude/Codex CLI가 추론. Yuminai는 그 위의 UI 레이어 |
| Claude Code의 직접 대체재 | **공존**. Claude Code를 자식 프로세스로 활용 |

## 시그니처 워크플로우 (디자이너가 가이드 메인 그림으로 쓸 수 있음)

```
(아침) 워크스페이스 nunchi-v2 클릭
   │
   ├─→ 어제 마지막 세션 자동 복원
   ├─→ Obsidian Daily Note 자동 컨텍스트
   │
(오전) 채팅창에 "PRD대로 옵션 B 흐름 구현, /tdd 먼저"
   │
   ├─→ Claude Code (자식 프로세스) Plan → Test → 구현
   ├─→ 출력은 Yuminai UI에 스트리밍
   │
(점심) 자리 비움 → Telegram 알림
   │  - "✅ 테스트 5/5 통과, 메인 코드 구현 진행 중"
   │
(외출) Telegram에 "PR 만들고 머지해줘" 답신
   │
   ├─→ 봇이 데스크톱 Yuminai에 명령 전달
   ├─→ Git PR 생성 + 머지 + 결과 알림
   │
(저녁) 작업 요약이 Obsidian Daily Note에 자동 append
```

## 기술 스택 (디자이너 참고용)

- **OS**: macOS 26 (Tahoe) 이상 *(macOS 26 신규 디자인 언어 활용)*
- **언어**: Swift 6.2 (strict concurrency)
- **UI**: SwiftUI 네이티브 (NSViewRepresentable 최소화)
- **DB**: SwiftData (로컬 영속)
- **시크릿**: macOS Keychain
- **핵심 외부**: Claude Code CLI · OpenAI Codex CLI · Obsidian Vault · Telegram Bot API · git CLI · gh CLI

## 현재 단계

- **버전**: v0.x (PRE-MVP, 본인 사용 중)
- **출시 계획**: 없음 (당분간 1인 사용)
- **공유 가능성**: 친한 개발자 1~2명에게는 향후 공유 가능
- **앱스토어**: 향후 가능성 (notarization 인프라 ADR-069에 준비됨)

## 가이드 톤 가이드

- **언어**: 한국어 (UI 자체가 한국어 — 영어 가이드도 가능하나 "기본은 한국어")
- **존댓말**: ~합니다 / ~해요 (반말 X, "공식적이지만 무겁지 않은")
- **이모지**: 자제. 헤딩/카테고리 아이콘에만 (✨ 🔧 등)
- **목소리**: 1인용 도구 — "본인의 작업 효율" 강조, "팀의 생산성" 같은 표현은 피함
- **분위기**: 미니멀 + 밀도 높음 + 다크 테마 친화 ("친절한 여백"보다 "한 화면 더 보이는 것")

## 핵심 슬로건 후보 (가이드 표지)

1. **"한 화면, 모든 코딩 사이클"** — 통합 가치 강조
2. **"Claude를 터미널 밖으로"** — 이전 도구 대비 차별
3. **"바이브 코딩, 정착하다"** — 컨셉 강조 ("바이브 코딩"은 사용자 본인의 단어)
4. **"사고는 노트로, 작업은 채팅으로, 결과는 Git으로"** — 워크플로우 스토리
5. **"모바일에서 명령, 데스크톱에서 실행"** — 모바일↔데스크톱 강조
